TC-UI-001
Verify user info section when the user is not logged in
Objective: Confirm the account header shows the correct anonymous state.
Precondition: User is logged out and has not stored an active session.
Steps:
  1. Open the My Account page.
  2. Observe the top user info section.
  3. Tap the login or sign-up entry points.
  • แสดง icon user ที่ตำแหน่งหัวข้อ user info
  • แสดงข้อความต้อนรับ “สวัสดี”
  • แสดงปุ่ม “เข้าสู่ระบบ” และ “สมัครสมาชิก” ให้ผู้ใช้กดได้
  • กดปุ่มแล้วต้องพาไปยัง flow ที่เหมาะสมสำหรับการเข้าสู่ระบบหรือสมัครสมาชิก
  • ไม่แสดงชื่อสมาชิกจริงหรือเมนูข้อมูลส่วนตัวของบัญชีที่ล็อกอินแล้ว
TC-UI-002
Verify user info section when the user is logged in as guest+
Objective: Confirm the guest+ account state exposes upgrade prompts and profile access.
Precondition: User is authenticated with a guest+ account.
Steps:
  1. Open the My Account page while logged in as guest+.
  2. Review the user info content and banner area.
  3. Select the profile entry point.
  • แสดง icon user เหมือนสถานะอื่น
  • แสดงข้อความเชิญชวนสมัครสมาชิก amaze แบบ guest+
  • แสดงปุ่ม “ข้อมูลส่วนตัว” ให้เข้าไปดูรายละเอียดโปรไฟล์
  • แสดง banner ชวนสมัคร amaze ตามดีไซน์ของหน้า
  • กดปุ่มข้อมูลส่วนตัวแล้วต้องเปิดหน้าหรือ drawer โปรไฟล์ที่ถูกต้อง
  • ไม่แสดงชื่อสมาชิกแบบเต็มเหมือนบัญชี amaze ที่ผูกโปรไฟล์แล้ว
TC-UI-003
Verify user info section when the user is logged in as amaze
Objective: Confirm the fully registered amaze account renders personalized account content.
Precondition: User is authenticated with an amaze account that has a saved profile.
Steps:
  1. Open the My Account page while logged in as amaze.
  2. Check the displayed name and the account menu list.
  3. Open profile, history, coupon, and help related entries.
  • แสดง icon user ที่ส่วนหัวของบัญชี
  • แสดงชื่อสมาชิกจริงจาก profile ที่ล็อกอินอยู่
  • แสดงปุ่ม “ข้อมูลส่วนตัว” เพื่อไปยังหน้ารายละเอียดบัญชี
  • แสดงเมนู order history, extra menu และ help section ครบถ้วน
  • แสดงเมนู happitat และ My Coupon ตามสิทธิ์ของบัญชี amaze
  • ข้อมูลที่แสดงต้องสอดคล้องกับ profile ของผู้ใช้ที่ล็อกอินใน session ปัจจุบัน
  • เมนูที่กดได้ต้องนำไปยังปลายทางที่ถูกต้องและไม่แสดงข้อมูลจากบัญชีอื่น

TC-UI-004
Verify order history section is shown for guest+ and amaze accounts
Objective: Confirm the order history block appears only for authenticated non-guest users.
Precondition: User is logged in as guest+ or amaze.
Steps:
  1. Open the My Account page with a guest+ account.
  2. Open the My Account page with an amaze account.
  3. Check whether the order history section and badge counters are visible.
  • แสดงหัวข้อ order history เฉพาะ guest+ และ amaze
  • แสดงปุ่ม “ดูทั้งหมด” หรือ see all สำหรับเข้าไปดูประวัติการสั่งซื้อ
  • แสดงเมนู To Ship, Shipping, To Receive, Return/Refund และ To Review ตามข้อมูล badge
  • กด “ดูทั้งหมด” แล้วต้องเปิดหน้า order history ที่แท็บ All
  • หากเป็นบัญชี guest จะไม่แสดง section นี้
TC-UI-005
Verify order history badge values and empty states
Objective: Confirm the badge counters render correctly from the fetched summary data.
Precondition: User is logged in and the order history summary API returns counts or null values.
Steps:
  1. Open the My Account page after the summary data is loaded.
  2. Inspect each order status icon badge.
  3. Verify hidden or null counts do not create broken badges.
  • ตัวเลข badge ต้องตรงกับ response ของแต่ละสถานะ
  • สถานะที่ไม่มีข้อมูลหรือเป็น null ต้องไม่แสดงตัวเลขผิด
  • การแสดง badge ต้องไม่ทำให้ layout แตกหรือซ้อนกับ icon
  • ค่า badge ต้องอัปเดตตาม session ปัจจุบัน ไม่ใช่ค่า cache ของผู้ใช้คนก่อน

TC-UI-009
Verify navigation and visibility matrix across all account levels
Objective: Confirm the page switches content correctly between guest, guest+, and amaze.
Precondition: Tester can switch account level or mock the stored session.
Steps:
  1. Open the page as guest.
  2. Open the page as guest+.
  3. Open the page as amaze.
  4. Compare header, banners, menu rows, and lower sections.
  • guest แสดงเฉพาะ login CTA, help section และ legal section
  • guest+ แสดง upgrade banner, order history, extra menu, affiliate, help และ law
  • amaze แสดง profile button, amaze banner, order history, extra menu, affiliate, happitat, help และ law
  • ทุกสถานะต้องไม่แสดง widget ของ state อื่นปนกัน
  • layout ต้องไม่เสียหายเมื่อสลับ account level
