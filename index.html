TC-UI-001
Verify user info section when the user is not logged in
Objective: Confirm the account header shows the correct anonymous state.
Precondition: User is logged out and has not stored an active session.
Steps:
  1. Open the My Account page.
  2. Observe the top user info section.
  3. Tap the login or sign-up entry points.
  • แสดง icon user ที่ตำแหน่งหัวข้อ user info
  • แสดงข้อความต้อนรับ “สวัสดี”
  • แสดงปุ่ม “เข้าสู่ระบบ” และ “สมัครสมาชิก” ให้ผู้ใช้กดได้
  • กดปุ่มแล้วต้องพาไปยัง flow ที่เหมาะสมสำหรับการเข้าสู่ระบบหรือสมัครสมาชิก
  • ไม่แสดงชื่อสมาชิกจริงหรือเมนูข้อมูลส่วนตัวของบัญชีที่ล็อกอินแล้ว
TC-UI-002
Verify user info section when the user is logged in as guest+
Objective: Confirm the guest+ account state exposes upgrade prompts and profile access.
Precondition: User is authenticated with a guest+ account.
Steps:
  1. Open the My Account page while logged in as guest+.
  2. Review the user info content and banner area.
  3. Select the profile entry point.
  • แสดง icon user เหมือนสถานะอื่น
  • แสดงข้อความเชิญชวนสมัครสมาชิก amaze แบบ guest+
  • แสดงปุ่ม “ข้อมูลส่วนตัว” ให้เข้าไปดูรายละเอียดโปรไฟล์
  • แสดง banner ชวนสมัคร amaze ตามดีไซน์ของหน้า
  • กดปุ่มข้อมูลส่วนตัวแล้วต้องเปิดหน้าหรือ drawer โปรไฟล์ที่ถูกต้อง
  • ไม่แสดงชื่อสมาชิกแบบเต็มเหมือนบัญชี amaze ที่ผูกโปรไฟล์แล้ว
TC-UI-003
Verify user info section when the user is logged in as amaze
Objective: Confirm the fully registered amaze account renders personalized account content.
Precondition: User is authenticated with an amaze account that has a saved profile.
Steps:
  1. Open the My Account page while logged in as amaze.
  2. Check the displayed name and the account menu list.
  3. Open profile, history, coupon, and help related entries.
  • แสดง icon user ที่ส่วนหัวของบัญชี
  • แสดงชื่อสมาชิกจริงจาก profile ที่ล็อกอินอยู่
  • แสดงปุ่ม “ข้อมูลส่วนตัว” เพื่อไปยังหน้ารายละเอียดบัญชี
  • แสดงเมนู order history, extra menu และ help section ครบถ้วน
  • แสดงเมนู happitat และ My Coupon ตามสิทธิ์ของบัญชี amaze
  • ข้อมูลที่แสดงต้องสอดคล้องกับ profile ของผู้ใช้ที่ล็อกอินใน session ปัจจุบัน
  • เมนูที่กดได้ต้องนำไปยังปลายทางที่ถูกต้องและไม่แสดงข้อมูลจากบัญชีอื่น
